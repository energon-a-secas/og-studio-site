This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
make serve          # Start dev server at http://localhost:8809 (ES modules require HTTP, not file://)
make kill           # Stop the dev server

# Regenerate pre-built OG images in assets/
npm install         # one-time: installs playwright devDependency
node scripts/generate-og.mjs
```

No build step, no lint, no test suite.

## Architecture

Modular ES module app. Entry point: `js/app.js`. All rendering is done via the Canvas API at 1200x630px.

**Two modes, one page:**
- **Gallery tab** — renders one canvas per site defined in `state.js:sites[]`, using a shared pattern + density setting. Supports individual and bulk PNG download.
- **Custom tab** — freeform canvas with sidebar controls for title, subtitle, accent color, gradient preset, pattern, density, and branding text.

**Data flow:**
1. `state.js` is the single source of truth: `sites[]` (all Neorgon tools with `id`, `title`, `subtitle`, `accent`, `domain`), `gradientPresets[]`, `patternList[]`, and the mutable `state` object (no localStorage persistence).
2. `render.js` builds the DOM grids and calls `patterns.js` drawing functions to paint each canvas.
3. `events.js` listens for control changes, updates `state`, and re-calls render functions.
4. `patterns.js` owns all canvas drawing: `drawGradientBg` → `drawPattern` → `drawVignette` → `drawText` → `drawBranding`. This layering order matters.

**Seeded RNG for determinism:** Patterns that use randomness (`constellation`, `circuits`, `waves`, `radial`, `noise`) use `mulberry32` seeded by hashing the site `id` string (or `title+subtitle` for custom mode). This ensures each site always renders the same pattern.

**Pattern dispatch:** `drawPattern(ctx, seedStr, accent, pattern, density)` in `patterns.js` maps density strings (`low`/`medium`/`high`) to a multiplier and calls the appropriate draw function.

**Pre-generated assets:** `assets/og-{site-id}.jpg` are static snapshots generated by `scripts/generate-og.mjs`, which spins up a local HTTP server, launches headless Chromium via Playwright, waits for `window.__ogStudioGalleryReady`, then exports each gallery canvas as JPEG (0.92 quality). Run `make generate` after changing site metadata, patterns, or adding new sites.

**Deploy:** `make deploy` (or `make og` = generate + deploy) runs `scripts/deploy-og.sh`, copying each `assets/og-{id}.jpg` to `{project-dir}/og-preview.jpg`. Site IDs in `state.js` must match the deploy map in `deploy-og.sh`.

## Key conventions

- Adding a new site: add an entry to `sites[]` in `js/state.js`, add `id:project-dir` to `DEPLOYS` in `scripts/deploy-og.sh`, then `make og`.
- Accent colors are appended with a 2-digit hex alpha for canvas operations (e.g., `accent + '30'` = 19% opacity). Keep accent values as 6-digit hex.
- The custom tab sidebar is rebuilt via `buildCustomCreator()` on init; it uses event delegation on the sidebar element rather than per-element listeners.
- `downloadCanvas` uses `canvas.toBlob` + a temporary `<a>` click — this works only in browser context, not in the generate script (which reads raw pixel data instead).
